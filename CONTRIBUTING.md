# Contributing to x402-Nexus

Thank you for your interest in contributing to x402-Nexus! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/x402-nexus.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Submit a pull request

## Development Setup

See [DEPLOYMENT.md](docs/DEPLOYMENT.md) for local development setup.

## Code Style

- Use TypeScript for all new code
- Follow existing code formatting
- Run `npm run lint` before committing
- Write tests for new features

## Pull Request Process

1. Update README.md with details of changes if needed
2. Update documentation in `/docs` if needed
3. Ensure all tests pass
4. Request review from maintainers
